也可能包含解析失败的中间代码(类似汇编)：

解析失败的代码多数是中间某个函数,可能是由于python小版本不同产生的不兼容问题。这些代码是dis模块将字节码反汇编后的结果，由字节码指令(Python Bytecode Instructions)组成。字节码指令的结构如下：

源码行号 | 跳转注释符 | 指令在函数中的偏移 | 指令符号（助记符） | 指令参数 | 实际参数值

注：上图中未显示跳转注释符，且省略了大部分指令参数

不同指令符号的意义可参阅文档：dis官方文档

其中：函数定义在第一行，参数是整体复刻后推测的；下一行指源码中第19行偏移量为0处字符串’http…’入栈，在偏移量为2处将str储存到FAST型变量中，名称为url；再下一行是源码中20行，全局量requests入栈,属性get入栈，接着’url’,header和cookies分别入栈，由字符串组成的常量tuple入栈，根据文档中的解释

可知CALL_FUNCTION_KW_3的作用机制：顶部元素是keyword组成的字符串元组，控制其下的关键字参数，因此’cookies’对应cookies，‘headers’对应headers；再下是位置参数url，底部是可调用的对象，也就是get。 即get(url, headers=headers, cookies=cookies) 最后将get的返回对象储存为res。如此根据文档和语法分析,我们就可以较为完整地还原python源码。

结语

如官方文档所提示的

CPython implementation detail: Bytecode is an implementation detail of the CPython interpreter. No guarantees are made that bytecode will not be added, removed, or changed between versions of Python. Use of this module should not be considered to work across Python VMs or Python releases.

由于不同版本的python的字节码指令规则不同，反编译的python版本的对应是非常关键的，否则可能产生大量类汇编代码，甚至完全无法反汇编。
